bsJS

최근 흥미를 가지고 공부를 하려고 하고 있는 것 중 하나가 javascript인데 마침 좋은 라이브러리 bsJS를 알게되서 공부도 할겸해서 bsJS 소스를 보기 시작했습니다. 처음 github에 가서 라이브러리를 다운받아보았더니 어라 1600줄 정도 되네요. 제가 javascript를 그냥 눈으로 보면서 읽고 이해할 수 없는 관계로 좋은 소스도 있겠다 평소 궁금했던 부분들을 bsJS는 어떻게 처리하고 있는지 알아보기로 했습니다. 아... 제가 보고 있는 소스가 최신은 아닙니다. 방금 확인했더니 또 바뀌어 있네요.

onload 시 자동 실행

내용보기 내용숨기기

  • jquery를 알기 전

    예전에 제가 jquery를 알기 전에는 전에는 아래와 같은 방법으로 실행했었습니다.
    body tag에 onload 를 걸어서 실행했었지요.
    그런데 body onload에서 스크립트를 호출하려니 불편합니다.

  • jquery를 알고난 후

    jquery 알게 된 후에는 $(function() { 실행할 명령 }) 이런 식으로 사용하게되었지요.
    js 파일을 여러개 나눠놓은 상태에서도 이 방법은 잘 실행이 되고, 심지어 여러 파일에서 각자 호출하면 각자 포함된 명령을 실행하니 편리해서 애용하고 있습니다.

  • bsJS 에서는

    그럼 bsJS에서는 어떻게 처리하고 있을까요? bsJS도 jquery와 비슷한 호출 방식을 사용하고 있습니다.
    bs( function(){ 실행할 명령 }) 과 같이 정의하면 실행이 됩니다.
    jquery와 마찬가지로 여러번 호출하면 각자 포함된 명령어를 실행해 줍니다.
    자! 그럼 소스를 봐봅시다. 1600 줄이넘는 소스라서 어디가 어디인지 모르겠습니다.
    대충 감으로 상관 없어보이는 소스들을 마구 삭제합니다.
    삭제하고 실행해보고 디버그하고 삭제하고 실행해보고 디버그하고를 몇번 반복했더니 아래와 같은 소스가 남습니다.
    이 남은 소스만 있으면 화면 오픈시 자동실행이 됩니다. 좋군요~ ^^
    살짝 주석을 달아봤는데 실력이 미천한지라 제대로 이해했는지는 모르겠습니다.

bsJS 기타 처리 내용

내용보기 내용숨기기

  • console 처리

    console이 없는 브라우저에서도 예외가 나지 않도록 console이 없는 경우에는 alert을 대신 호출하도록 정의되어 있습니다.

  • Array 확장

    Array의 불편함을 해소하기 위해 prototype에 indexof를 구현했습니다.

  • Date.Now

    Date.Now 가 존재하면 사용
    존재하지 않으면 new Date를 반환해줍니다.

  • JSON parse, stringfy

    요즘 최신 브라우저에는 거의 다 JSON처리 기능을 가지고 있습니다.
    그런데 과거 브라우저에는 JSON처리하는 내용이 없을수도 있습니다.
    여기서 bsjs의 철학일까라요? 고객은 브라우저 모른다! 없어도 나오게 해주자! 입니다.
    그래서 윈도우 객체를 확인해보고 JSON처리하는 기능이 존재하지 않으면 parse와 stringify함수를 정의해 주고 있습니다.
